<a name="data-protection-consumer-apis-dangerous-unprotect"></a>

<span data-ttu-id="d9467-104">Le API di protezione dati ASP.NET Core non sono principalmente destinati indefinita persistenza del payload riservati.</span><span class="sxs-lookup"><span data-stu-id="d9467-104">The ASP.NET Core data protection APIs are not primarily intended for indefinite persistence of confidential payloads.</span></span> <span data-ttu-id="d9467-105">Come altre tecnologie [Windows CNG DPAPI](https://msdn.microsoft.com/library/windows/desktop/hh706794%28v=vs.85%29.aspx) e [Azure Rights Management](/rights-management/) sono più adatte allo scenario di archiviazione illimitata, e hanno le funzionalità di gestione delle chiavi sicuro o ridotta di conseguenza.</span><span class="sxs-lookup"><span data-stu-id="d9467-105">Other technologies like [Windows CNG DPAPI](https://msdn.microsoft.com/library/windows/desktop/hh706794%28v=vs.85%29.aspx) and [Azure Rights Management](/rights-management/) are more suited to the scenario of indefinite storage, and they have correspondingly strong key management capabilities.</span></span> <span data-ttu-id="d9467-106">Ciò premesso, non c'è niente divieto di uno sviluppatore di usare le API di protezione dati ASP.NET Core per la protezione a lungo termine dei dati riservati.</span><span class="sxs-lookup"><span data-stu-id="d9467-106">That said, there's nothing prohibiting a developer from using the ASP.NET Core data protection APIs for long-term protection of confidential data.</span></span> <span data-ttu-id="d9467-107">Le chiavi non vengono mai rimossi dal Keyring, pertanto `IDataProtector.Unprotect` sempre possibile recuperare i payload esistenti, purché le chiavi sono disponibili e validi.</span><span class="sxs-lookup"><span data-stu-id="d9467-107">Keys are never removed from the key ring, so `IDataProtector.Unprotect` can always recover existing payloads as long as the keys are available and valid.</span></span>

<span data-ttu-id="d9467-108">Tuttavia, un problema si verifica quando lo sviluppatore prova a rimuovere la protezione dei dati che sono stato protetto con una chiave revocata, come `IDataProtector.Unprotect` genererà un'eccezione in questo caso.</span><span class="sxs-lookup"><span data-stu-id="d9467-108">However, an issue arises when the developer tries to unprotect data that has been protected with a revoked key, as `IDataProtector.Unprotect` will throw an exception in this case.</span></span> <span data-ttu-id="d9467-109">Questo può essere adatto per i payload di breve durati o temporanei (ad esempio, i token di autenticazione), perché questi tipi di payload possono essere ricreati facilmente dal sistema e nel peggiore dei casi i visitatori del sito potrebbe essere necessario accedere di nuovo.</span><span class="sxs-lookup"><span data-stu-id="d9467-109">This might be fine for short-lived or transient payloads (like authentication tokens), as these kinds of payloads can easily be recreated by the system, and at worst the site visitor might be required to log in again.</span></span> <span data-ttu-id="d9467-110">Ma per i payload persistenti, visto `Unprotect` throw può causare la perdita di dati inaccettabile.</span><span class="sxs-lookup"><span data-stu-id="d9467-110">But for persisted payloads, having `Unprotect` throw could lead to unacceptable data loss.</span></span>
